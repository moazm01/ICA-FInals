01 / Prerequisites

The In-Order Pipeline Bottleneck

⚠️ The RAW Data Hazard

In a standard 5-stage RISC pipeline, instructions execute sequentially. High-latency instructions cause cascading stalls down the pipeline.

MUL.D F0, F2, F4 ; 4 cycles ADD.D F6, F0, F8 ; STALLS waiting for F0 SUB.D F8, F12, F14 ; BLOCKED despite NO dependencies!

💥 Pipelining Performance Loss

  • Structural Stalls: Pipeline freezes entirely.
  • Wasted Cycles: 18 stall bubbles in standard 7-inst benchmark.
  • In-Order CPI: ~5.3 cycles per instruction (ideal is 1.0).
02 / Concept

Tomasulo's Solution: Out-of-Order Execution

1. Reservation Stations

Buffers in front of execution units holding pending instructions and operand values/tags.

2. Register Renaming

Eliminates WAR and WAW hazards by mapping register names to reservation station tags.

3. Common Data Bus

Broadcasts completed results directly to all waiting reservation stations in parallel.

04 / Hardware Formalism

Moore FSM & VHDL Implementation

⚙️ RS as a Moore State Machine

Four discrete states where outputs depend solely on the current state:

IDLE ➔ WAITING ➔ EXECUTING ➔ WRITEBACK ➔ IDLE

05 / Engine Design

Reverse Pipeline Cycle Execution

⏱️ Clock Edge Accuracy: Write ➔ Execute ➔ Issue

The simulation engine executes stages in reverse order each cycle to faithfully model hardware clock boundaries. Results broadcast at cycle N are captured at cycle N+1.

def step_cycle(self): self.clock += 1 self._write_result_stage() # 1. Broadcast CDB results self._execute_stage() # 2. Count down latency self._issue_stage() # 3. Dispatch next instruction
